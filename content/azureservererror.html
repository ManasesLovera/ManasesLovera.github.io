Azure server error 500.30

Well... I wanted to talk about this because it was one of the first errors I couldn't find a solution the first time, I didn't find the solution in the documentation, stackoverflow, youtube, google, chatgpt... I was stuck and didn't know what to do, the backend project was working perfectly locally but when and publish the project on Azure I was getting this generic error for 3 days.

But, then I discovered it was not that hard, I was just calling something from the appsettings.json file and wasn't using it, and yeah, sounds stupid, 3 days and I just needed to not using it and delete it or just use it, in that momment I was implementing push notifications to the project and I just did the base configuration.

If you ever get this error the FIRST thing you must do is click on the project in Azure Portal -> App Services, then click on "Diagnose and solve problems", look for the Diagnostic Tools section and click on "Application Event Logs", this may sound extremely obvious for most developers with some experience but since I just published a backend project without any previous knowledge I had no idea how that was working, the first time I used it I had to publish the project, I was scared and had no idea about anything, I asked for help but my mind was stuck, sometimes the way you feel in a job may block you for being productive.

Below I will attach the docs and other sites you can find info about this problem, I know I just showed one situacion where you find this issue and there is a lot of situations you face this, but please know you are not alone, take a deep breath and ask for help, most of the times it's not that hard, you are making it harder.
