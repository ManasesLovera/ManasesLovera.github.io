Data Annotations vs Fluent Validation: Object Validation in .NET

I have an internal debate about which is better for validating DTOs. This is the conclusion I reached: Both allow you to validate equally and provide the same tools. Both are equally easy to check for validation once everything is configured. However, FluentValidation takes more time to configure since it must be done via DependencyInjection for each validator, and an external library must be installed. Data Annotation: csharp Copiar código if(!ModelState.IsValid) return BadRequest(ModelState); FluentValidation: csharp Copiar código var validationResult = await validator.ValidateAsync(Dto); if (!validationResult.IsValid) { return BadRequest(validationResult.Errors); } Data Annotations are easier to implement, while FluentValidation requires another file, installation, dependency injection, and more code for validation. In FluentValidation, when the data is invalid, errors come as an array with everything that is invalid. In contrast, in Data Annotations, it is an object with each property, and each one comes with an array with the name of each property. I prefer how FluentValidation handles this part, as it is less complicated for the frontend. Considering that code should be written for both your understanding and for others to read, FluentValidation offers a better way to view all the validations that the DTO will have to be valid without having to review each DTO's data annotations. In my opinion, although Data Annotations, being an internal library, are much easier to implement in your code, FluentValidation can allow the developer to understand more easily, quickly, and accessibly, and provide better-organized error handling. It is worth taking a few extra minutes to install, etc. I hope you find this useful!
